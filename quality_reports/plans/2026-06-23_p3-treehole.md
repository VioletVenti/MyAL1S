## Context（为什么做这个）

P2 把「可控写操作」竖切（交作业）打通并预留了 `auto` 权限级别 + 语义组机制供「文件无关写」复用。roadmap 的 P3 = 外部论坛。经调研 + 用户决策，P3 聚焦 **北大树洞**（选课写暂缓——其写需验证码/ttshitu，方案未定）。

树洞价值：① 读——把校内匿名讨论区接进仪表盘（浏览/关注/我的/历史）；② 写——发帖/回复，是 P2 预留 `auto` 级别的首个真实用例（文件无关写）。

**协议事实（2026-06-23 实测抓取，非猜测，已存 memory `myal1s-p3-treehole-protocol`）：**
- 域名 `treehole.pku.edu.cn`，根 → `/ch/web/`（Vite SPA）。
- **IAAA appid = `PKU Helper`**（树洞属 PKU Helper app 生态）。
- 认证链：`GET /chapi/redirect_iaaa_login?version=3&uuid=<UUID>&plat=web` → 302 到 `iaaa.pku.edu.cn/iaaa/oauth.jsp?appID=PKU Helper&redirectUrl=…/chapi/cas_iaaa_login?…` → 回调 → `/api/v6/iaaa_callback` → **app-JWT**（`access_token` + `/api/v6/refresh_token`）。
- ⚠️ 树洞认证 = `oauth.jsp`(JS 登录页) + app-JWT，**不同于** pku3b 现有的 `oauthlogin.do` POST + 服务端 ssoLogin.do 写 cookie。`oauth.jsp` 经 P2 OTP 经验证明**不是**可 HTTP 重放的免 OTP 捷径。**能否用 pku3b 现有 IAAA 客户端换到树洞 JWT，是最大未验证点 → 先 spike。**
- 读端点（`/api/v3`）：`hole/list`、`hole/get`、`hole/list_comments`、`hole/my_list`、`hole/history`、`hole/attention`。
- 写端点：`hole/post`（发帖）、`comment/post`（回复）、`hole/attention*`（关注）、`hole/praise`/`tread`（赞踩）。
- 信封：JSON，成功 `code === 20000`（bundle 里 `t.code===2e4`）。

## Locked decisions（grilling 锁定）

1. P3 = 树洞读写；选课写暂缓。
2. **认证 spike 先行**（pku3b Rust 模块，复用 cfg.toml + 用户输 OTP），证明能拿到树洞 JWT。
3. **spike 失败 → 继续拱树洞**（换策略：手动抓 cookie 填入 / 逆向 oauth.jsp JS），不 fallback 到选课。
4. 读 MVP = `hole/list` + `hole/get` + `hole/list_comments` + `hole/attention`(关注) + `hole/my_list`(我的) + `hole/history`(历史)。
5. 发帖默认 **confirm**（走 P2 审批横条）；本轮**实现** `auto` 分支（点亮 P2 预留 else），但 treehole_post 默认仍 confirm，用户手动提级才自主。
6. agent = 读工具直调 + 写工具复用 P2 计划1 审批（语义组 `treehole_post`，无文件）。

## 复用点（已确认，不重造）

- **pku3b IAAA 半段**：`iaaa_oauth_login(appid,…)` / `iaaa_is_mobile_authen`（`low_level/iaaa.rs:54,107`）——app 无关，传 `"PKU Helper"`。
- **服务半段模板** = `thesis_lib`（bearer token 模型，`low_level/thesis_lib.rs` + `api/thesis_lib.rs`），不是 portal（cookie 模型）。
- **JSON 取数** = blackboard `api_get<T: DeserializeOwned>`（`low_level/blackboard.rs:304`）。
- **MCP 工具** = `tool_specs()` + `ToolRegistry::call` dispatch + `otp_optional_schema` + `ok`/`needs_otp`；`auth::login_treehole` 镜像 `auth::login_portal`（`mcp/auth.rs:59`）。
- **P2 闸复用**：`PermissionGate._dispatch` 加 treehole 分支；`KNOWN_GROUPS` 加 `treehole_post`；`_VALID_LEVELS` 加 `auto`（点亮 else）；agent 工具集 `FilteredToolset` 不需改（treehole 写工具是本地 FunctionToolset，非 path 原语）。
- **前端**：`frontend/src/Dashboard.tsx` 的 `北大树洞` DeferredPanel 占位（`deferred` 数组）变真实面板；TreeholePost 类型已在 `api.ts`。

## 实施增量（S 是硬门；A/B/C 在 S 成功后）

### Increment S — 树洞认证 spike（pku3b，硬门）`[RISK: 最高]`
**目标**：证明 `oauth.jsp → app-JWT` 交换可行，拿到真 token 并调通一次 `/api/v3/hole/list`。
- 新建 `pku3b/src/api/low_level/treehole.rs`：URL 常量（`TREEHOLE_APP_ID="PKU Helper"`、`redirect_iaaa_login`、`cas_iaaa_login`、`iaaa_callback`、API base）+ `impl LowLevelClient { treehole_login_require_otp, treehole_login(→TreeholeToken), treehole_api_get<T> }`。
- 新建 `pku3b/src/api/treehole.rs`：`impl Client { async fn treehole(user,pass,otp) -> Treehole }` + `struct Treehole { client, token }` + 一个 `list_holes()` 验证调用。
- 在 `api/mod.rs` + `api/low_level.rs` 注册 `pub mod treehole;`（无 feature flag，无条件，仿 portal/syllabus）。
- **spike 驱动**：新建 `pku3b/src/cli/cmd_treehole.rs`（仿 `cmd_syllabus`）一个 `Login`/`Probe` 子命令——用户在终端跑 `pku3b treehole probe`、inquire 输 OTP、打印是否拿到 token + hole/list 首屏。复用现有 CLI 的 cfg.toml + inquire OTP 流。
- **不确定步骤**：`oauth.jsp` 是 JS 页，`RequestBuilder::send()` 不执行 JS。spike 要探明：IAAA token 拿到后，是直接 GET `cas_iaaa_login?…&token=` 返回 JSON JWT，还是要多跳。可能是 ~15 行 redirect-follow（像 portal）或多步 scrape（像 thesis_lib）——**探明即 spike 产出**。
- **验证**：`cargo test`（单测 URL/解析）+ 用户跑 `pku3b treehole probe` 实测拿 token + list。
- **分支**：成功 → 进 A。失败 → 按决策 3 换策略（手动 cookie / 逆向 JS），不进 A，本轮在 spike 内迭代。

### Increment A — 树洞读（MVP）`[RISK: 中]`

- 前端：`Dashboard.tsx` 的 `北大树洞` DeferredPanel 占位换成真实 `TreeholePanel`（列表 + 分页 + 点击进详情/楼层）。类型 `TreeholePost` 已在 `api.ts`（按真响应补字段）。

### Increment B — 树洞写 + 点亮 auto `[RISK: 中高]`
**依赖**：A。点亮 P2 预留的 `auto` 分支。
- pku3b：`api/treehole.rs` 加 `post_hole`/`post_comment`（写，调 `/api/v3/hole/post`、`/comment/post`）；`mcp/tools.rs` 注册 `treehole_post`/`treehole_comment`（`read_only:false`，path-free 纯 JSON 参数）——这些是**执行原语**，经 `gateway.call_tool` 派发。
- 后端 `PermissionGate`：`KNOWN_GROUPS` 加 `treehole_post` + `treehole_comment`；`_WRITE_TOOLS` 加两名；`_dispatch` 加 treehole 分支（直接转发 JSON args，无 file_id 解析）；**`_VALID_LEVELS` 加 `auto`** + `level_for`/`create_approval`/`execute_now` 的 else 分支：`auto → 直接 _dispatch`（不经 pending）。默认仍 confirm（决策 5）。
- agent：本地 FunctionToolset 工具 `treehole_post(text, …)` → `gate.create_approval(group="treehole_post", …)`；复用 P2 审批横条。无文件，故不需 FilteredToolset 隐藏（无 path 原语）。
- 测试：gate 单测加 `auto` 级别（提级后直接执行不 pending）+ treehole post 派发（假 gateway）。

## Risks & 取舍
- **最大风险**：S 的 oauth.jsp→JWT 交换（spike 前无法预知形态）。已用 spike 门控，失败不波及 A/B/C。
- 树洞可能不在选课/教学网的 IAAA 信任设备链里——一次 OTP 能否同时连树洞需 S 验证（memory 已记此点）。
- 读响应字段需 S/A 拿真响应后补类型（bundle 只给端点不给字段）。
- auto 首次启用：默认 confirm 兜底，防「替你公开发帖」。
- 选课写本轮不做（验证码方案未定）。
